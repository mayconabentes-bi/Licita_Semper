# LicitaSemper - Execution Rules

To maintain high code quality and stability, the following rules must be observed:

1. **Strategic Planning**: Before modifying multiple files, present a short plan outlining the changes.
2. **Atomic Commits**: Implement features in small, logical steps.
3. **Ambiguity Resolution**: Ask objective questions before coding if requirements are unclear.
4. **Architectural Consistency**: Do not assume stack or infrastructure details; always verify against the repository's configuration.
5. **Deterministic Logic**: Prefer clear, deterministic business logic for core calculations (like the Scoring Motor) unless AI intervention is explicitly requested.
